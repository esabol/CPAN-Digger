CPAN Digger

The goal of CPAN Digger is to provide a tool to help you find ways to improve CPAN modules. Originally it started with the list of packages most recently uploaded to CPAN. The thought behind that was that it is a lot more likely for a Pull-Reqest to be accepted in a project that recently saw activity than something that has not been changed for a long time.
As an experiement we can show the releases of selected CPAN authors, or information about a list of releases supplied in a file. If you'd like to be added to the list open an issue asking to be added. If you'd like to generate your own report you can do it by cloning the CPAN-Digger repository and running it locally.
A number of issues we try to encourage people to fix.
  • Make sure the META data includes a link to the public VCS (Version Control System) of the project. This will let MetaCPAN display the link and it will make it easy for any potential contributor to find the VCS (git, mercurial, etc.) repository of the project. It will make it easier to contribute to the project.
  • Explicit link to the bug-tracker of the project. If it does not exist in the META-files of the project, MetaCPAN will link to RT by default even if the project has a GitHub repository with Issues enabled. Even if you would like to use RT, it would be better to state this explicitely in the META files.
  • CI - Most project have test and Perl has the amazing CPAN Testers, but developers can get feedback to their source code from a Continous Integration (CI) system much faster. They can avoid releasing broken versions. So we would like to encourage setting up CI for each project.
  • Have a valid license key in the META files will make it easier for a process to verify that all the packages in a dependency meet the licensing requirements of the project.
  • CPAN Cover runs the tests of many CPAN uploads and collects the test coverage report of them. Here include the percentages here with a link to the full report. We would like to encourage people to contribute tests to open source Perl projects thereby increasing their quality.
